# handoff/ — пакет для Claude Code

Это всё, что нужно отдать агенту-кодеру (Claude Code), чтобы он понял, **что именно** надо реализовать и **как именно** это должно выглядеть.

## Что внутри

```
handoff/
├── README.md       ← ты здесь
├── PROMPT.md       ← готовый первый промпт. Скопировать → вставить в Claude Code
├── SPEC.md         ← бриф: архитектура страницы, контракт API, темизация, что НЕ копировать
├── tokens.css      ← все CSS-переменные (цвета, типографика, тени, радиусы) отдельным файлом
└── screenshots/
    ├── 01-dark.png   …   06-dark.png    ← эталон в тёмной теме (6 скроллов)
    └── 01-light.png  …   06-light.png   ← эталон в светлой теме
```

Основным файлом-эталоном остаётся **`../Atman Demo Visual Upgrade.html`** в корне проекта — там и hi-fi мок целиком, и патчи Python внизу, и скрипт переключения темы. Всё в этой папке — справочные материалы вокруг него.

## Как пользоваться

1. Открой `PROMPT.md`, скопируй блок с промптом.
2. В Claude Code: создай новый чат в нужном репозитории, приложи к сообщению:
   - сам файл `Atman Demo Visual Upgrade.html`
   - всю папку `handoff/` (или хотя бы `SPEC.md` + `tokens.css` + пару скриншотов)
3. Вставь промпт. В одном месте промпта (`Шаблонизатор: <ВПИШИ — …>`) подставь свой стек (Jinja / FastAPI templates / vanilla / etc).
4. Получив первую версию — открой dev-сервер и сравни глазами со скриншотами `03-dark.png` / `03-light.png` (там основной tab-content с inputs + outputs + report). Расхождения скорми Claude Code «бонус-промптом» из конца `PROMPT.md`.

## Что Claude Code НЕ должен делать

- Тащить в проект `.doc-header`, `.changelog`, `.patches`, `.browser-bar`, `.pill-tag v2 · gradio 6` — это docs-обёртка вокруг мока, в проде её нет.
- Использовать `react-json-tree` для JSON-output — это плоский `<pre>` с подсветкой классов.
- Делать три отдельных карточки вместо одной `.report-group` для Detailed Analysis Report.
- Выпиливать эмодзи 🔥 ▶️ 🚧 🔍 📊 📑 🌐 ⚡ 🧠 — они являются якорями секций.
