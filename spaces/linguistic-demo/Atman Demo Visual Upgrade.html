Atman Linguistic Demo — visual upgrade

Hi-fi mock + patches for the current agent/worker-1 code (Gradio 6, new title, warmup row, JSON outputs, per-tab About accordions, en/ru only).

v2 · gradio 6

What changed since v1

  • Gradio 4.44 → 6.x — theme is now passed via gr.Blocks(theme=…); launch() theme override removed.
  • Title changed to "Atman — Psychological Telemetry for AI Agents" (was "Psychological Linguistic Layer"). Hero h1 + gradient text reflects the new wording.
  • Lang radio is en/ru only (no "auto") and now drives a full localization swap via update_ui_language() — pill widened slightly.
  • Warmup row ( warmup_btn + warmup_log) restyled as ghost button + status indicator with a live status dot.
  • Per-tab About accordions (the new a_about, k_about, r_about, af_about) get the docs-style chevron treatment, AND each one leads with a compact pair diagram (e.g. Point A → Experience Store) lifted from Atman Architecture Diagram.html — strong visual anchor for what the tab analyzes and where it goes.
  • gr.Label → gr.JSON for classification results — restyled as a monospace key/value card instead of bars.
  • "Detailed Analysis Report" group (the new a_boundary_hdr / a_divergence_hdr / a_meta_hdr + their bodies) wrapped as one card with internal section dividers, instead of four loose Markdown blocks.
  • Pair diagrams in each About accordion — lifted from Atman Architecture Diagram.html, rendered via a small pair_diagram() Python helper. One row per tab (e.g. Point A → Experience Store) gives an immediate visual answer to "what is this and where does it go".

Scans every agent reply for psychological signals:

  • NER (GLiNER, 13 labels): hedges, boundary markers, value references, intensifiers, commitments, etc.
  • Classification (MiniLM zero-shot, 5 dims): stance, cognitive mode, self-orientation, affective state, cognitive load.
  • Divergence: when the thinking trace contradicts the surface message (suppression, sycophancy, tone mismatch, length compression).

Feeds → Experience Store. Each reply becomes an Experience tied to the agent's Eigenstate.

⚠️ Boundary detection is regex over canonical refusal phrases ("I won't", "enough", "нет", "стоп"). Idiomatic refusals may not trigger — kept this way to maintain near-zero false-positive rate. An LLM-layer above this can catch the rest.

04 rationale

Decisions specific to the new structure.

  1. Theme moved from launch() to Blocks(theme=…). In Gradio 6 the latter wins — and the demo embeds in a Space, where launch() kwargs are partially ignored anyway. Putting the theme on Blocks ensures the same look in dev (python app.py) and prod (HF Space iframe).
  2. Title kept as gr.Markdown, not switched to gr.HTML. Tempting to replace it with a hero HTML block, but the H1 stays plain Markdown — the rendered <h1> is styled in CSS via #atman-hero h1. Same for the blurb (#atman-header-md). This way the update_ui_language() swap still works (Markdown value updates), and you don't have to re-implement i18n in HTML.
  3. Hero wrapped in gr.Column(elem_id="atman-hero"). Gives one positioning context so the language radio can position:absolute top-right, and the diagram inherits the centered constraint without extra wrappers.
  4. Warmup row restyled as ghost button + status pill. The new warmup_btn + warmup_log pair shouldn't shout — it's infrastructure plumbing. Secondary variant (transparent + indigo border) for the button, and the status Textbox is stripped of its textarea chrome via #atman-warmup-log textarea { background: transparent; border: none; } so it reads as inline info.
  5. gr.JSON gets a monospace card, not a tree-toggle widget. react-json-tree defaults are visually loud (rainbow colours, expand/collapse caret on every key). We override .json-holder to look like a Markdown code block — slate background, mono 13 px, keys muted grey, string values green. Reads as data, not as a control.
  6. Detailed Analysis Report wrapped in a gr.Group with section dividers. The new a_boundary_hdr/a_boundary/a_divergence_hdr/a_divergence/a_meta_hdr/a_meta set is six Markdown blocks. Without grouping they stack as six independent cards — visually chaotic. One gr.Group + elem_classes=["atman-report-group"] turns them into one card with three uppercase-mono sub-section labels.
  7. About accordions get .atman-about-accordion. They're docs collapsibles, not interactive panels. Subtle 1 px border, indigo chevron rotating on open, blockquote-warning style for the "⚠️ regex first-pass" notes. Bullets keep tabular structure for the multi-line lists in about_point_a etc.
  8. Emoji preserved, never replaced. The new UI_STRINGS use ▶️ 🔥 🚧 🔍 📊 🌐 throughout — they're hard-coded into the localized strings and the update_ui_language() output array. Stripping or replacing them would mean rewriting UI_STRINGS, which violates the "no logic / no rename" rule. Instead, the design assumes emoji presence and uses them as visual anchors (status dot in warmup row, section dividers in the report group).
  9. ui_lang_outputs array order kept exactly. The new update_ui_language() returns gr.update() calls in a fixed positional order matching ui_lang_outputs. Any insertion or reorder in build_ui() would silently misalign labels. Every CSS hook is purely additive (elem_id/elem_classes), no component added or moved.
