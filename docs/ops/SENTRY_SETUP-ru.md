# Настройка Sentry

> **English version:** [SENTRY_SETUP.md](SENTRY_SETUP.md)

## Обзор

Интеграция Sentry в Atman полностью opt-in. Если `SENTRY_DSN` не задан, модуль наблюдаемости является полным no-op — SDK не загружается, сетевые вызовы не выполняются, накладные расходы на runtime равны нулю.

Интеграция находится в `src/atman/adapters/observability/sentry.py`.

## Конфигурация

Задайте переменную окружения `SENTRY_DSN` для включения Sentry:

```bash
SENTRY_DSN=https://<key>@sentry.io/<project-id>
```

Дополнительные опциональные переменные:

```bash
# Тег окружения в Sentry (dev | routine | ci)
SENTRY_ENVIRONMENT=dev

# Доля транзакций для записи в мониторинге производительности (0.0–1.0)
SENTRY_TRACES_SAMPLE_RATE=0.1
```

## Что отслеживается

При включённом Sentry адаптер захватывает:

| Сигнал | Детали |
|--------|--------|
| **Ошибки** | Необработанные исключения из всех компонентов Atman |
| **Routine spans** | Выполнение периодических maintenance и фоновых задач |
| **Ошибки трансляции** | Ошибки при конвертации между доменными моделями и представлениями адаптеров |
| **Транзакции сессий** | Полный жизненный цикл сессии от `start_session` до `finish_session` |
| **Maintenance spans** | Выполнение отдельных maintenance-задач (claim → dispatch → complete) |

## Окружения

Используйте `SENTRY_ENVIRONMENT` для тегирования событий контекстом их происхождения:

| Значение | Когда использовать |
|----------|-------------------|
| `dev` | Локальная разработка и ручное тестирование |
| `routine` | Запланированные фоновые задачи (daily рефлексия, maintenance) |
| `ci` | Автоматизированные тесты в CI-пайплайнах |

Разные окружения позволяют фильтровать шум в интерфейсе Sentry — например, исключать `ci`-события из production-дашбордов.

## Просмотр данных

1. Откройте свой проект Sentry на [sentry.io](https://sentry.io).
2. Перейдите в **Issues** для просмотра ошибок, сгруппированных по типу.
3. Перейдите в **Performance** для просмотра трассировок транзакций.
4. Используйте фильтр **Environment** для переключения между `dev`, `routine` и `ci`.

## Версия SDK

Atman требует `sentry-sdk >= 2.35.0`. Эта версия указана как опциональная зависимость и устанавливается автоматически при использовании extras `sentry`:

```bash
pip install "atman[sentry]"
```

Или установите вручную:

```bash
pip install "sentry-sdk>=2.35.0"
```

Если SDK не установлен, а `SENTRY_DSN` задан, Atman выведет предупреждение при запуске, но продолжит работать в штатном режиме.
