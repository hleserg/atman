# Session Reports

Directory for session-linked reports and baselines.

## Naming

Use filenames like:

- `YYYY-MM-DD_HHMM_session-<session-id>.md`

## Required header

Each report should include:

- timestamp
- session id / session key
- topic
- key metrics
- decisions / follow-ups

## Purpose

These reports are meant to make it obvious which session they belong to and when they were created.
